01

作用域的定义与理解

作用域

定义：当前执行的上下文。其中的值和表达式是"可见的"，或者是可以引用的。

JS中的作用域

  • 全局作用域: 在Web中为window, Node中为global​​​​​​​ 
  • 函数作用域​​​​​​​​​​​​​
  • 块级作用域

块级作用域

定义：任何一对花括号中的语句集都属于一个块，在这之中定义的所有变量在代码块外都是不可见的，我们称之为块级作用域。

块声明： { }

02

变量提升

作用域的问题:变量提升

变量提升（Hoisting）被认为是， Javascript中执行上下文（特别是创建和执行阶段）工作方式的一种认识。

变量提升详解

从概念的字面意义上说，“变量提升”意味着变量和函数的声明会在物理层面移动到代码的最前面，但这么说并不准确。实际上变量和函数声明在代码里的位置是不会动的，而是在编译阶段被放入内存中。

函数提升的说明

function isEven(n) {
  if(n === 0) {
    return true;
  }
  return isOdd(n - 1)
}

function isOdd(n) {
  if(n === 0) {
    return false
  }
  return isEven(n - 1)
}

var let const

var声明

通过var声明的变量没有块级作用域。换句话说，语句块不会生成一个新的作用域 

var x = 1; 
{
 var x = 2; 
} 
console.log(x); // 输出 2

var可以重复声明

var a = 1;
var a = 'ster';

var全局声明

在全局作用域声明成为全局的属性

var a = 1; 
console.log(window.a) //1 
console.log(delete window.a) //false
b = 3
console.log(window.b) // 3
console.log(delete window.b) //true

var全局声明的隐藏问题

修改全局的预定义函数

Math.abs(-1); // 1

var  Math = 2;

Math.abs(-4);
// Uncaught TypeError: Math.abs is not a function

let声明

let语句声明一个块级作用域的本地变量，并且可选的将其初始化为一个值。 

{
  let a = 1, b;
}

暂时性死区（temporal dead zone）

let或const声明的变量拥有暂时性死区（TDZ）：当进入它的作用域，它不能被访问（获取或设置）直到执行到达声明才可以被访问。var声明的变量不具有暂时性死区

TDZ

if (true) { 
tmp = 'abc'; // ReferenceError
console.log(tmp); // ReferenceError
let tmp; // TDZ ends, `tmp` is initialized with `undefined`
console.log(tmp); // undefined
tmp = 123;
console.log(tmp); // 123
}

TDZ

if (true) { // enter new scope, TDZ starts
    const func = function () {
        console.log(myVar); // OK!
    };
    // Here we are within the TDZ and
    // accessing `myVar` would cause a `ReferenceError`
    let myVar = 3; // TDZ ends
    func(); // called outside TDZ
}

typeof 和TDZ

一个变量在TDZ中是无法访问的，这意味着不能用typeof来判断它的类型

if (true) { 
console.log(typeof tmp); 
// => ReferenceError: tmp is not defined 
let tmp = 'abc'; 
}

let与var的相同点

let声明的变量只在其声明之后块区域或子块中可用，这一点，与var相似。 

function letTest() {
  let x = 1;
  if (true) {
    console.log(x);  // 1
  }
  console.log(x);  // 1
}

let与var的区别

var声明的变量的作用域是整个封闭函数，let只能是声明之后的作用域

function varTest() {
  console.log(x);  // undefined
  var x = 1;
  console.log(x);  // 2
}

function letTest() {
  console.log(x);  // Uncaught ReferenceError: x is not defined
  let x = 1;
  console.log(x);  // 1
}

let不可重复声明

var a;
var a = 1;
let a = 1;
let a = 3; //Identifier 'a' has already been declared

let与var的区别

在全局作用使用let定义的时候并不会成为全局变量的属性

let a = 1;
console.log(window.a) //undefined

const常量

const​​​​​​​ 声明创建一个值的只读引用 ，其作用域可以是全局或本地声明的块。需要一个常数的初始化器；也就是说，您必须在声明的同一语句中指定它的值 ​​​​​​​

const与for循环

const a = [1, 2, 3]
for(const test of a){
  console.log(test)
}
// 1
// 2
// 3

const b = {
  name: 'lisi',
  age: 23,
  sex: 'male'
}
for(const key in b){
  console.log(key)
}
// name
// age
// sex

const的变与不可变

JavaScript中有5中基础数据类型，分别是Undefined、Null、Boolean、Number、String。使用const声明后不可改变，数组,对象使用const声明后可以改变里面的内容 

const a = '1';
a = 2; //Uncaught TypeError: Assignment to constant variable.
const b = [1, 2, 3];
b[1] = 10; //ok
const c = {
  name: 'zhangsan'
};
c.name = 'lisi'; //ok

关于JavaScript的内存机制

与C++类似,JavaScipt中的内存也分为栈区和堆区,它们存储数据的机制不同,导致存储数据的类型也不同,栈区主要存储简单数据类型,堆区主要存储引用数据类型.​​

引用类型占据空间大、大小不固定 ,栈内存中存放地址指向堆(heap)内存中的对象。是按引用访问的

关于函数参数作用域

let y = 1;
function foo(x = y, y) {
    console.log(x);
}
foo(); // ReferenceError: y is not defined

函数作用域解读

  1. 当函数存在默认参数时，且调用方法不传任何参数，会存在三个作用域环境；​​​​​​​
  2. 全局作用域、参数作用域、函数体作用域
  3. 当执行foo 函数时，参数作用域在x = y 之后才定义let y，注意：let 定义，所以根据let 定义变量的作用知道x = y 肯定会报错；
